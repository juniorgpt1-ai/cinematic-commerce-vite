Acertos

O site apresenta excelente arquitetura técnica (code-splitting, CSS-inline fallback, fontes auto-hospedadas, animações com prefers-reduced-motion), mas sofre de 4 problemas críticos que afetam SEO e acessibilidade. Os itens P0 devem ser corrigidos antes do lançamento.

⚡ 1. Performance (Core Web Vitals) ♿ 2. Acessibilidade (WCAG 2.1) 🎨 3. Design (Tipografia / Cores / Layout) 🔍 4. SEO

⚡ 1. Performance (Core Web Vitals)

✅ O Que Está Certo

OK
Hero Fallback Inline (Anti-CLS)
index.html contém um fallback HTML estático do hero (#hero-fallback) com MutationObserver que esconde automaticamente quando o React hidrata. Isso elimina CLS durante o carregamento inicial — técnica exemplar.
OK
LCP Image Preload com media queries
Dois no com fetchpriority="high" — um para desktop (malbec-signatureA.webp) e outro para mobile (malbecSMOB.webp), ambos com atributo media para evitar download duplo.
OK
Code-splitting agressivo
12 seções com React.lazy() + Suspense, mais AntigravityParticles e Toaster. Manual chunks no vite.config.ts agrupam seções por contexto (haircare, perfumes, products, bottom).
OK
Fontes auto-hospedadas com preload
Fonts em /fonts/ como WOFF2, com no head. Sem dependência de Google Fonts CDN — zero round-trips externos.
OK
CSS Preload Plugin customizado
vitePluginCssPreload() em vite.config.ts transforma a stylesheet em não-bloqueante (media="print" onload="this.media='all'") com fallback
OK
Imagens com width/height explícitos
Imagens na Home possuem atributos width e height (ex: 1600×1200, 2400×1792), prevenindo layout shifts durante o carregamento lazy.
OK
Cache-Control headers robustos no Express
server/index.ts define max-age=31536000, immutable para assets com hash (js, css, woff2, imagens) e max-age=0, must-revalidate para HTML.

⚠️ Pontos de Atenção (P1)

P1
framer-motion ainda no bundle
Muitas animações já foram migradas para CSS puro (reveal-up, animate-fade-up, etc.), mas framer-motion permanece como dependência e é empacotado no chunk vendor-framer. Verificar se ainda há imports residuais de motion ou AnimatePresence em componentes. Se não houver, remover a dependência economiza ~30 KB gzipped.
P1
Fontes não utilizadas carregadas
fonts/ contém 6 arquivos WOFF2 (~300 KB total): fenix-400.woff2, lora-400.woff2, playfair-display-600.woff2, cormorant-garamond.woff2, plus-jakarta-sans.woff2, plus-jakarta-sans-italic.woff2. O font-display CSS declara Playfair Display e Cormorant Garamond, mas Fenix e Lora não são referenciados — remover do bundle.
P1
modulePreload desabilitado para vendor-framer
O resolveDependencies em vite.config.ts filtra vendor-framer dos module preloads. Se framer-motion não for mais usado, essa lógica pode ser removida. Se ainda for usado, module preload seria benéfico.
P1
Sem cabeçalho Content-Security-Policy
O server/index.ts define X-Content-Type-Options, X-Frame-Options e Referrer-Policy, mas não inclui CSP. Recomendável adicionar para proteção contra XSS.

💡 Melhorias (P2)

P2
Preload de fontes incompleto
Apenas 2 das fontes usadas (Cormorant Garamond e Plus Jakarta Sans) têm preload. Playfair Display, usada em headings de seções, não é pre-carregada — potencial atraso de renderização de headings abaixo da dobra.
P2
AntigravityParticles com requestAnimationFrame
O canvas de partículas é inicializado com requestAnimationFrame após o primeiro frame. Isso é bom, mas o componente em si (Three.js ou canvas) pode ser pesado. Considerar um loading="lazy" mais agressivo — talvez carregar apenas quando o hero estiver visível há >2 segundos.
P2
Sem compressão Brotli no Express
O Express server em server/index.ts não usa middleware de compressão. Com Vercel isso é tratado automaticamente, mas para deploy standalone, adicionar compression middleware com Brotli seria benéfico.

🔴 Crítico (P0)

P0
Contraste insuficiente: texto dourado sobre fundo claro
--luxe-gold: #9A7B50 sobre --luxe-bg: #faf9f6
#9A7B50
#faf9f6 3.28:1
Falha WCAG AA para texto normal (mínimo 4.5:1). Passa apenas para texto grande (≥18px ou ≥14px bold). Afeta: .eyebrow, --color-luxe-gold-deep (#7A5E3A = 4.77:1 — OK), text-luxe-gold-soft (#B89A6A = 2.57:1 — ainda pior).
Correção: Escurecer --luxe-gold para #7A5E3A ou #6B4F2E, ou usar apenas em textos ≥18px bold.
P0
Ausência de skip-to-content link
Usuários de teclado/leitor de tela precisam navegar por toda a Nav (desktop ou mobile) antes de chegar ao conteúdo principal. Um link "Pular para o conteúdo" no topo da página é exigido por WCAG 2.4.1 (Bypass Blocks).
P0
Ausência de landmark
A página usa
como wrapper (Home.tsx linha 41), mas não declara explicitamente role="main" ou usa
que já carrega landmark implícito. Verificar: na Home.tsx, o elemento é
— isso está correto! Porém, o fallback #hero-fallback no index.html está fora de qualquer landmark, o que pode confundir screen readers antes da hidratação.

⚠️ Pontos de Atenção (P1)

P1
Carrossel sem navegação por teclado
O carrossel (BoticarioCarousel, MalbecShowcase) oferece botões de seta com aria-label, mas não responde a ArrowLeft/ArrowRight do teclado quando o foco está no container. Também não possui região aria-live para anunciar mudanças de slide a leitores de tela.
P1
Conteúdo oculto do FAQ acessível no DOM
As respostas do FAQ usam grid-rows-[0fr] + overflow-hidden para colapsar. O conteúdo colapsado permanece no DOM e pode ser lido por screen readers mesmo quando aria-expanded="false". Idealmente usar inert ou aria-hidden="true" no conteúdo colapsado, ou remover do DOM com display:none após a transição.
P1
Textos com baixo contraste sobre imagens
text-white/70 sobre overlay gradiente escuro no hero — a relação de contraste varia com a imagem de fundo. Embora tenha drop-shadow como reforço, o texto descritivo pode ser ilegível em alguns trechos da imagem. O overlay from-black/95 via-black/70 mitiga parcialmente.

🎨 3. Design (Tipografia / Cores / Layout)

✅ O Que Está Certo

OK
Hierarquia tipográfica bem definida
3 famílias com papéis claros: Cormorant Garamond (display — hero, títulos), Playfair Display (section headings), Plus Jakarta Sans (body, navegação). Cada uma configurada em @theme inline como --font-display, --font-section, --font-sans.
OK
Sistema de cores robusto com CSS custom properties
Paleta "luxe" com 10 variantes de gold (soft, deep, light) + ink/ink-soft + bordo + line + glow-rgb. Cores base em oklch() para perceptually uniform contrast. Design tokens bem organizados.
OK
CSS utilities reutilizáveis
Sistema de utilities com @utility: eyebrow, gold-rule, card-premium, btn-gold-metallic, glass-btn, bg-luxe-gradient, wa-glow, wa-pulse, scrollbar-premium. Isso garante consistência visual e reduz CSS repetido.
OK
Animações de reveal padronizadas
Sistema de .reveal-up, .reveal-right, .reveal-scale com .visible acionado por IntersectionObserver via useScrollReveal(). .stagger-container com nth-child delays. Consistência exemplar.
OK
Responsividade mobile-first
Breakpoints md: (768px) e lg: (1024px). Nav completamente adaptada para mobile (header premium com 3 linhas: nav links, brand title, brand logos). Imagens com + source srcSet para mobile vs desktop. Hero com clamp() para tipografia fluida.
OK
Micro-interações refinadas
btn-hover-scale com cubic-bezier(0.22, 1, 0.36, 1), wa-glow pulsante no CTA WhatsApp, SendMorphIcon animado no clique, FloatingBadge com bob, dot indicators com progress ring no carrossel. Toque premium consistente.

⚠️ Pontos de Atenção (P1)

P1
Dark mode preparado mas não implementado
@custom-variant dark (&:is(.dark *)) está definido e o ThemeProvider tem suporte a switchable (comentado). As variáveis :root estão prontas, mas não há toggle de tema na interface. Seria um diferencial de UX.
P1
Imagens JPEG residuais
A raiz de client/public/ contém floratta.jpeg (provavelmente não usado, já que existe floratta.webp). Remover assets não referenciados reduz o tamanho do deploy.

💡 Melhorias (P2)

P2
Text wrap inconsistente no Hero
O título do Hero usa leading-[0.85] no mobile e md:leading-[1.0] no desktop. O line-height muito apertado (0.85) pode causar clipping de descendentes (como 'g', 'p', 'ç') em algumas fontes/tamanhos. Testar em dispositivos reais.
P2
Estados de loading dos componentes lazy
Todos os Suspense usam fallback={null}. Isso significa que durante o carregamento lazy, o espaço fica vazio (sem skeleton ou spinner). Para seções abaixo da dobra, aceitável. Para TrustBar (primeira seção após hero), poderia causar layout shift percebido.
